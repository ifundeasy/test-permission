Write an ADR for `$ARGUMENTS` in English, using the repo's ADR format if one exists (else a standard
template): Title; Status (proposed/accepted/superseded); Context (forces and constraints);
Decision (what and why); Alternatives considered (with trade-offs); Consequences (positive, negative,
follow-ups). Store under the repo's ADR/docs location with a sequential number. Author only —
do not implement the decision. Do not commit.
